# Supabase セットアップ手順

このアプリのバックエンド（データの保存場所）を **Supabase** で用意する手順です。
各自が自分の Supabase プロジェクトを作り、最後に `.env` に接続情報を貼って完了です。

> このファイルは「Supabase側の準備」だけをまとめたものです。
> Flutterのインストールやアプリの起動方法は [README.md](../README.md) を見てください。

---

## 必要なもの

- Supabase アカウント（無料・**クレジットカード不要**）

---

## 手順は4つだけ

1. プロジェクトを作る
2. データの入れ物を作る（SQLをコピペ）
3. メール確認をOFFにする
4. 接続情報を `.env` に貼る

---

## ステップ1：プロジェクトを作る

1. [supabase.com](https://supabase.com) にログイン
2. 「New project」を押す
3. プロジェクト名は自由
4. リージョン（地域）は **Tokyo (Northeast Asia)** がおすすめ
5. データベースのパスワードは適当でOK（あとで使いません）
6. 作成完了まで数分待つ

---

## ステップ2：データの入れ物を作る（コピペするだけ）

1. 画面左メニューの **SQL Editor** を開く
2. このアプリの中の [supabase/schema.sql](../supabase/schema.sql) を開いて、**中身を全部コピー**
3. SQL Editor に貼り付けて **Run（実行）** を押す

これ1回で、次が全部できます。

- **テーブル**（users / groups / group_members / posts / post_shares）
- **RLS**（誰がどのデータを読み書きできるかのルール）
- **Storage バケット**（動画とアイコンの置き場：videos / icons）とアクセス権

> ⚠️ `schema.sql` は「最初の1回だけ」実行してください。
> 再実行すると既存データを消して作り直します。

---

## ステップ3：メール確認をOFFにする（重要・忘れがち）

Supabase は初期設定だと、新規登録時にメール確認が必須です。
これがONのままだと **サインアップしてもログインできず詰まります。** 必ずOFFにします。

1. 画面左メニューの **Authentication** を開く
2. **Sign In / Providers**（または Settings）を開く
3. **Email** の設定にある **Confirm email** を **OFF** にする

> これはSQLでは設定できない、画面上での操作です。

---

## ステップ4：接続情報を `.env` に貼る

1. アプリのフォルダで設定ファイルの雛形をコピー

   ```bash
   cp .env.example .env
   ```

2. Supabaseの画面で **Project Settings → API** を開く
3. 次の2つをコピーして `.env` に貼る

   ```
   SUPABASE_URL=（Project URL を貼る）
   SUPABASE_ANON_KEY=（anon public キーを貼る）
   ```

> 🔑 `.env` は GitHub に上げません（設定済み）。

---

## これで完了

あとは [README.md](../README.md) の手順でアプリを起動すれば、自分のSupabaseにつながった状態で動きます。

---

## 困ったときは

| こんな症状 | こうする |
|------------|----------|
| 起動したらエラー画面が出る | `.env` の URL・キーを確認（ステップ4） |
| 新規登録したのにログインできない | メール確認のOFFを忘れている（ステップ3） |
| 動画のアップロードに失敗する | `schema.sql` を実行したか確認（ステップ2） |

---

## まとめ：Supabaseのいいところ

- `.env` に貼る値は **2つだけ**（URL と anon key）
- データの入れ物は **SQLコピペ1回** で完成
- **クレジットカード不要** で Auth・データベース・動画保存まで全部使える
