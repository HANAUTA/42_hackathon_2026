# 並列開発キックオフ

STEP2を4つのClaude Codeインスタンスで並列開発するための手順書。
全体設計は [開発計画.md](./開発計画.md)、進捗は [進捗管理.md](./進捗管理.md) を参照。

---

## worktree構成

各レーンは独立した worktree（作業ディレクトリ）で動かす。互いに衝突しない。

```
/Users/yanase/hnut/42_hackathon_2026  [dev]         ← 統合・マージ担当
/Users/yanase/hnut/setlog-auth        [feat/auth]   → Lane A 認証
/Users/yanase/hnut/setlog-home        [feat/home]   → Lane B ホーム
/Users/yanase/hnut/setlog-group       [feat/group]  → Lane C グループ
/Users/yanase/hnut/setlog-post        [feat/post]   → Lane D 投稿
```

各 worktree には `.env`（Git管理外）をコピー済み。

---

## 事前準備（1回だけ・テストデータ投入）

1. Supabase → Authentication → Users → Add user で自分のアカウント作成（メール確認OFF推奨）
2. そのユーザーのUUIDをコピー
3. [supabase/seed.sql](../supabase/seed.sql) の `v_user` にUUIDを入れて SQL Editor で Run

→ テストグループA/B・投稿3件が入る。Lane B/C/D はこれで他レーンを待たず開発できる。

```bash

cd /Users/yanase/hnut/setlog-group && flutter pub get && claude   # Lane C
cd /Users/yanase/hnut/setlog-post  && flutter pub get && claude   # Lane D
```

起動したら、下のキックオフプロンプトを各インスタンスに貼る。

## 並列の鉄則（全インスタンス共通）

| ルール | 理由 |
|--------|------|
| 自分の `features/xxx/` だけ触る | ファイル衝突を防ぐ |
| `core/router.dart`・`models/` は変更前に相談 | 共有ファイル |
| ポートは Lane ごとに分ける（8766〜8769） | 同時起動の衝突防止 |
| 各自 `flutter pub get` を最初に実行 | worktreeごとに `.dart_tool` が必要 |
| ログインユーザーは `supabase.auth.currentUser` で取得 | Lane Aの完成を待たず開発できる |
