# Firebase セットアップ手順

このアプリのバックエンド（データの保存場所）を **Firebase** で用意する場合の手順です。
各自が自分の Firebase プロジェクトを作り、最後に接続情報を `.env` に貼って完了です。

> このファイルは「Firebase側の準備」だけをまとめたものです。
> Flutterのインストールやアプリの起動方法は [README.md](../README.md) を見てください。

> ⚠️ **注意：このアプリは現在 Supabase で動いています。**
> この手順書は「もし Firebase に置き換えたら」を試すための比較用メモです。
> 実際に動かすにはアプリ側のコード（認証・DB・動画保存）の書き換えも必要です。

---

## 必要なもの

- Google アカウント（Firebase はこれでログイン）
- **クレジットカード**（動画保存＝Storage を使うために必要。下の注意を参照）

---

## 手順は5つ

1. プロジェクトを作る
2. ログイン機能（Auth）を有効にする
3. データベース（Firestore）を作る
4. 動画の置き場（Storage）を有効にする ← ここでクレカが必要
5. 接続情報を `.env` に貼る

---

## ステップ1：プロジェクトを作る

1. [Firebase コンソール](https://console.firebase.google.com) にログイン
2. 「プロジェクトを追加」を押す
3. プロジェクト名は自由
4. Google アナリティクスは「無効」でOK（ハッカソンでは不要）
5. 作成完了まで少し待つ

---

## ステップ2：ログイン機能（Auth）を有効にする

1. 左メニューの **構築 → Authentication** を開く
2. 「始める」を押す
3. ログイン方法の一覧から **メール / パスワード** を選び、**有効** にして保存

> Supabaseと違い、メール確認は初期状態でOFFなので、ここは特別な操作は不要です。

---

## ステップ3：データベース（Firestore）を作る

1. 左メニューの **構築 → Firestore Database** を開く
2. 「データベースを作成」を押す
3. ロケーションは **asia-northeast1（東京）** がおすすめ
4. 最初は **テストモード** で作成（あとでルールを締める）

> Firestore は「スキーマレス」なので、Supabaseのような **テーブル作成SQLは不要**です。
> アプリがデータを書き込めば、入れ物は自動でできます。

---

## ステップ4：動画の置き場（Storage）を有効にする

1. 左メニューの **構築 → Storage** を開く
2. 「始める」を押す
3. ロケーションは Firestore と同じ **asia-northeast1** にする

> ⚠️ **クレジットカードが必要です。**
> 2024年後半から、新しいプロジェクトで Cloud Storage を使うには
> **Blaze プラン（従量課金）への切り替え＝クレカ登録**が必要になりました。
> 無料の Spark プランのままでは Storage が使えません。
> （無料枠の範囲内ならほぼ課金は発生しませんが、カード登録自体は必須です）

---

## ステップ5：接続情報を `.env` に貼る

1. Firebase コンソールで **プロジェクトの設定（歯車アイコン）** を開く
2. 「マイアプリ」で対象プラットフォーム（Web / Android / iOS）を追加し、表示される設定値を控える
3. アプリのフォルダで設定ファイルの雛形をコピー

   ```bash
   cp .env.example .env
   ```

4. `.env` に次の値を貼る（Supabaseより数が多い）

   ```
   FIREBASE_API_KEY=（apiKey）
   FIREBASE_APP_ID=（appId / プラットフォームごとに違う）
   FIREBASE_PROJECT_ID=（projectId）
   FIREBASE_MESSAGING_SENDER_ID=（messagingSenderId）
   FIREBASE_STORAGE_BUCKET=（storageBucket）
   FIREBASE_AUTH_DOMAIN=（authDomain / Web用）
   ```

> 🔑 `.env` は GitHub に上げません（設定済み）。
> ⚠️ `APP_ID` は Web / Android / iOS でそれぞれ別の値です。使うプラットフォームの分を貼ります。

---

## 困ったときは

| こんな症状 | こうする |
|------------|----------|
| 起動したらエラー画面が出る | `.env` の値に貼り間違いがないか確認（値が多いので注意） |
| Storage が有効にできない | Blaze プラン（クレカ登録）への切り替えが必要（ステップ4） |
| ログインできない | Authentication でメール/パスワードを有効にしたか確認（ステップ2） |

---

## まとめ：Supabase と比べたときの違い

| | Firebase | （参考）Supabase |
|---|---|---|
| `.env` に貼る値 | **6つ前後**（APP_IDがOS別） | 2つだけ |
| データの入れ物 | 不要（スキーマレス） | SQLコピペ1回 |
| 動画保存にクレカ | **必要（Blaze）** | 不要 |
| メール確認の設定 | 不要（初期OFF） | OFFにする操作が必要 |

- **手軽さ・クレカ不要を重視するなら Supabase が有利。**
- Firebase は「データの入れ物を作らなくていい」点は楽だが、**クレカ必須**と**設定値の多さ**がハッカソン配布ではネックになりやすい。
