# 課題集（ハッカソン参加者向け）

このアプリ（Setlog風 動画共有アプリ）のMVPをベースに、機能を追加する課題集。
難易度を3段階に分けている。初級は全員、中級以上から選択、を想定。

> 運営向けの事前準備・要決定事項は末尾の「運営メモ」を参照。

---

## 難易度の考え方

| レベル | 定義 | 触る範囲 |
|--------|------|---------|
| **初級** | バックエンドを触らない。Flutterのwidget / 状態 / アニメだけで完結する | フロントのみ（触っても既存テーブルの1カラムUPDATE止まり） |
| **中級** | 新しいテーブルを1つ追加し、リレーション・集計・RLSポリシーを書く | フロント + DB設計 |
| **上級** | 横断的・インフラ寄り。Edge Function / 通知 / リアルタイム / 既存ロジックの一般化 | フロント + DB + サーバー機能 |

初級と中級は**地続き**になるよう設計している（例: 「ハートが飛ぶ演出（初級・見た目）」→「いいね永続化（中級・DB保存）」）。
次のレベルに進む実感を得やすい。

各課題の構成: **概要 / 合格条件（デモで見せる動き） / ヒント（触るファイル） / 発展**

---

## 初級 — UIのみ・バックエンドなし

目安: 90分〜半日。Flutterのwidget・状態・アニメだけで完結する。

### ◎ 課題1: 未投稿枠の「動くニコちゃん」

- **概要**: グループ詳細で、まだ投稿していないメンバーの枠は現在「砂時計アイコン＋まだ投稿していません」の静止表示。これを、本家のように動くニコちゃんマーク（揺れる・弾むなど）に差し替える。
- **合格条件**: 未投稿の空き枠で、ニコちゃんがアニメーションで動いている。
- **ヒント**:
  - 対象は `_EmptyMemberCard`（[lib/features/group/group_detail_screen.dart](../lib/features/group/group_detail_screen.dart) 内）。`Icons.hourglass_empty` の箇所を置き換える。
  - `AnimationController` + `AnimatedBuilder` / `TweenAnimationBuilder` で揺れ・弾みを作る。
- **発展**: 複数枠でアニメの位相をずらす／タップで反応する。

### ◎ 課題2: 送信プレビューにスタンプ・フレームを重ねる

- **概要**: 撮影後の送信プレビューで、動画の上にスタンプ（絵文字/PNG）やフレームを重ねて飾れるUIを作る。**動画ファイル自体は加工しない**（再生に重ねて表示するだけ）。
- **合格条件**: プレビュー再生中、選んだスタンプ／フレームが動画の上に表示される。
- **ヒント**:
  - 送信プレビュー周り（[lib/features/post/send_screen.dart](../lib/features/post/send_screen.dart)、video_preview_factory_*）。
  - `Stack` で `video_player` の上に `Positioned` でスタンプを重ねる。ドラッグ移動は `GestureDetector` + 座標state。
  - ⚠️ 動画ファイルへの「焼き込み（書き出し）」は上級領域なので、初級ではやらない。
- **発展**: スタンプのドラッグ移動・拡大縮小・回転。

### 課題3: ダブルタップで「ハートが飛ぶ」演出（いいねの見た目だけ）

- **概要**: 投稿カードをダブルタップするとハートがふわっと飛ぶアニメを出す。**保存はしない**（見た目だけ）。
- **合格条件**: 投稿をダブルタップするとハートのアニメが再生される。
- **ヒント**: `GestureDetector(onDoubleTap:)` + `AnimatedOpacity` / `AnimatedScale`。
- **発展**: → 中級「いいね機能」でこの操作をDB保存に接続する。

### その他の初級候補

| 課題 | 中身 |
|------|------|
| ダークモード / テーマ切替 | `ThemeData` の切替 |
| アイコン未設定時のイニシャルアバター | 名前から色＋頭文字を生成して表示 |
| 時間移動のスワイプ対応 | グループ詳細の時間移動を `PageView` 化 |

## 中級 — 新テーブル + リレーション + RLS

目安: 半日〜1日。DB設計を1つ自分でやる。

### ◎ 課題4: いいね機能

- **概要**: 投稿にいいねを付けられるようにする。件数表示と、自分が押したかの状態保持。
- **合格条件**: 投稿にいいねでき、件数が表示され、自分のいいね状態がリロード後も保持される。
- **ヒント**:
  - 新テーブル `likes(id, post_id, user_id, created_at)` を追加。`UNIQUE(post_id, user_id)` で二重いいね防止。
  - RLSポリシー（読み取り全員 / 作成・削除は本人）を `supabase/schema.sql` の既存ポリシーに倣って書く。
  - 件数は集計、自分の状態は `user_id` で判定。
- **発展**: 初級「ハートが飛ぶ演出」を、この保存処理に接続する。

### 課題5: コメント機能

- **概要**: 投稿にコメントを付けて一覧表示する。
- **合格条件**: 投稿にコメントを投稿でき、コメント一覧が表示される。
- **ヒント**: 新テーブル `comments(id, post_id, user_id, body, created_at)`。1対多のリレーション・一覧UI・投稿フォーム。RLSは `likes` と同様。

### その他の中級候補

| 課題 | 中身 |
|------|------|
| 投稿削除 | 自分の投稿を削除（posts / post_shares のdelete + Storage削除） |
| グループ脱退 | `group_members` のレコード削除 + 画面更新 |
| グループ内の集計表示 | 今日の投稿者数・時間帯別投稿数などを集計して表示 |

## 上級 — 横断的・インフラ・複雑な状態

目安: 1日〜。1つ作れれば十分。

### 課題6: 新着投稿のプッシュ通知

- **概要**: 同じグループに新しい投稿があったら、他のメンバーに通知を送る。
- **合格条件**: あるメンバーが投稿すると、同グループの他メンバーの端末に通知が届く。
- **ヒント**: Supabase Edge Function + FCM。投稿挿入をトリガに通知送信。
- **⚠️ 注意**: Webでは動かない。**APK（Android実機）前提**の課題。

### 課題7: 未読管理（バッジ表示）

- **概要**: 未視聴の投稿数をグループにバッジで表示する。
- **合格条件**: 未視聴投稿があるグループにバッジ（件数）が出て、見ると消える。
- **ヒント**: `last_seen` 的な管理 + リアルタイム購読（Supabase Realtime）。

### 課題8: 投稿制限のグループ別設定

- **概要**: 現在「1グループにつき一定時間に1回」の投稿制限を、**グループごとに間隔を設定できる**ように一般化する。
- **合格条件**: グループ設定で制限間隔（例: 1時間 / 15分 / なし）を選べ、送信画面のグレーアウト判定に反映される。
- **ヒント**: 現在の制限ロジックは `shared_hour` / `shared_date` ベース（[lib/features/post/post_provider.dart](../lib/features/post/post_provider.dart)、`supabase/schema.sql` のUNIQUE制約）。これをグループ設定値で駆動するよう一般化する。
- **備考**: 運営が事前に実施する「15分への変更」と地続き。事前実装をお手本コードとして読ませると理解が早い。
