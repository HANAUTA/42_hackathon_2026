# 運営セットアップ（GitHub Actions + Firebase App Distribution）

push したら自動で APK がビルドされ、参加者のスマホに届く仕組みの準備手順です。
**運営メンバーが1回だけやればOK。** 参加者はこの作業をする必要はありません。

---

## 全体像

```
コード push → GitHub Actions が APK を自動ビルド
  → Firebase App Distribution にアップロード
    → 参加者のスマホに通知 → タップしてインストール
```

---

## 1. Firebase プロジェクトを作る

1. [Firebase コンソール](https://console.firebase.google.com) にログイン
2. 「プロジェクトを追加」→ プロジェクト名は自由（例: `setlog-hackathon`）
3. Google アナリティクスは「無効」でOK

---

## 2. Android アプリを登録する

1. プロジェクトの概要画面で **Android アイコン** をクリック
2. パッケージ名: `com.setlog.setlog_app`
3. アプリのニックネーム: `Setlog`（任意）
4. `google-services.json` のダウンロードは **スキップでOK**（配布にしか使わないので不要）
5. 残りのステップも「次へ」→「コンソールに進む」で完了

---

## 3. App Distribution を有効にする

1. 左メニューの **リリースとモニタリング → App Distribution** を開く
2. 「始める」を押す

---

## 4. テスターグループを作る

1. App Distribution の画面で **「テスターとグループ」** タブを開く
2. **「グループを追加」** → グループ名を `testers` にする
3. 参加者のメールアドレスを追加する

> 参加者には Firebase から招待メールが届きます。
> 参加者は「承認」して「Firebase App Tester」アプリを入れるだけです。

---

## 5. サービスアカウントのキーを取得する

GitHub Actions から Firebase にアップロードするための認証キーです。

1. [Google Cloud コンソール](https://console.cloud.google.com) を開く
2. 上部のプロジェクト選択で、ステップ1で作った Firebase プロジェクトを選ぶ
3. 左メニュー **IAM と管理 → サービスアカウント** を開く
4. **「サービスアカウントを作成」** をクリック
   - 名前: `github-actions`（任意）
   - ロール: **Firebase App Distribution 管理者**
5. 作成したサービスアカウントの **「鍵」タブ → 「鍵を追加」→ JSON** を選ぶ
6. JSON ファイルがダウンロードされる（この中身をあとで GitHub に登録する）

---

## 6. Firebase App ID を確認する

1. Firebase コンソール → **プロジェクトの設定**（歯車アイコン）
2. 「マイアプリ」の Android アプリの **アプリ ID** をコピーする
   - `1:123456789:android:abcdef` のような形式

---

## 7. GitHub Secrets に登録する

GitHub リポジトリの **Settings → Secrets and variables → Actions → New repository secret** で、以下の4つを登録します。

| Secret 名 | 値 |
|---|---|
| `SUPABASE_URL` | Supabase の Project URL（例: `https://xxx.supabase.co`） |
| `SUPABASE_ANON_KEY` | Supabase の anon public キー |
| `FIREBASE_APP_ID` | ステップ6でコピーしたアプリID |
| `FIREBASE_SERVICE_ACCOUNT` | ステップ5でダウンロードした JSON ファイルの**中身をそのまま貼る** |

---

## 8. 動作確認

1. 適当な変更を `main` に push する
2. GitHub リポジトリの **Actions** タブでワークフローが動いているか確認
3. 緑のチェックがついたら、テスターのスマホに通知が届いているか確認

> 初回はFlutterのセットアップで **5〜8分** かかります。
> 2回目以降はキャッシュが効いて少し速くなります。

---

## トラブルシューティング

| 症状 | 対処 |
|---|---|
| Actions が赤くなる（ビルド失敗） | Actions のログを開いてエラー内容を確認。Secrets の値が間違っていることが多い |
| ビルドは成功したのに通知が来ない | Firebase App Distribution にテスターのメアドが追加されているか確認 |
| テスターが招待メールを受け取れない | 迷惑メールフォルダを確認。Firebase App Tester アプリを先に入れてもらう |
| `FIREBASE_SERVICE_ACCOUNT` のエラー | JSON の中身をそのまま貼ったか確認（ファイルパスではなく中身） |
