# 必須課題

まずはチーム全員で、この **3つの必須課題** に取り組みます。
アプリの基本的な開発の流れ（画面の編集 → データの読み書き → 新しい画面の追加）を一通り体験できる内容です。

> 💡 各課題は別々の人が分担してもOK。終わったら [自由課題](自由課題.md) に進みましょう。

---

## 1. ログイン画面のUIを改善する

**レベル：かんたん（Flutterのみ）**

ログイン画面の見た目を、自分たちのチームらしく整えます。
色・文字・余白・ロゴなどを変えるだけでもOK。**バックエンドは触りません。**

- 対象ファイル：`lib/features/auth/` 以下
- やること例：背景色やボタンの色を変える／タイトル文言を変える／ロゴや画像を置く

> 🎯 ねらい：Flutter のウィジェットと見た目の編集に慣れる。

---

## 2. グループから脱退する

**レベル：ふつう（DB操作・API理解）**

参加中のグループから抜ける機能を作ります。
ボタンを押すと、自分の参加情報（`group_members`）が削除されてグループ一覧から消える、という流れです。

- 関連テーブル：`group_members`（`group_id` と `user_id` で1行に対応）
- やること例：グループ画面に「脱退」ボタンを追加 → 押したら Supabase から自分の行を削除 → 一覧を更新

> 🎯 ねらい：Supabase へのデータ削除（DELETE）と、画面の再読み込みの流れを理解する。

---

## 3. プロフィール画面を作り、名前を編集できるようにする

**レベル：ふつう（新しい画面の追加）**

新しく **プロフィール画面** を作り、自分の表示名（`users.name`）を編集・保存できるようにします。

- 関連テーブル：`users`（`name`、`bio` など）
- やること例：プロフィール画面を新規作成 → ルーティングに追加（`core/router.dart`）→ 名前を入力して保存 → Supabase の `users` を更新

> 🎯 ねらい：画面の新規追加・ルーティング・既存データの更新（UPDATE）を通しで体験する。

---

## 進め方のヒント

- わからないことは **Claude Code に聞く**のが一番早いです（「`group_members` から自分の行を削除する Dart コードを書いて」など）。
- 既存の似た画面（`lib/features/group/`、`lib/features/post/`）を参考にすると書きやすいです。
- コードを書くときは [コーディング規約](コーディング規約.md) に沿って書きましょう。
- テーブルの構造は [データベース設計](データベース設計.md) を参照。
