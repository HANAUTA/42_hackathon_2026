# 自由課題

[必須課題](必須課題.md) が終わったチームは、ここから好きな課題を選んで挑戦しましょう。
初級・中級・上級の3レベルに分かれています。

---

## 課題設計の考え方

今回のハッカソンでは、基本的にこちらで用意したSupabase環境を使って開発します。

そのため、初級〜中級の課題では **新しいテーブルを追加しなくても実装できるもの** を中心にします。

Flutter側の見た目変更、既存データの表示、既存カラムへの更新などを通して、まずはアプリ開発に慣れてもらうことが目的です。

一方で、より本格的にバックエンドやDB設計にも触れたい人向けに、上級として **自分たちのSupabaseを作るコース** を用意します。

上級コースでは、こちらで用意したSQLを実行して、必要なテーブル構成を自分たちのSupabaseに複製します。

その上で、likesテーブル、commentsテーブル、reactionsテーブルなどを使う課題に挑戦してもらいます。

## レベル分けの意図

### 初級

Flutterだけで完結する課題。

DBや認証に触れず、まずは画面変更・アニメーション・状態管理に慣れる。

### 中級

こちらで用意したSupabaseを使う課題。

新しいテーブルは作らず、既存のデータを読む・更新する・集計する体験をする。

### 上級

自分たちでSupabase環境を作る課題。

SQLを実行してテーブルを用意し、DB設計・リレーション・リアルタイム更新まで触れる。

---

## 上級コースの流れ

1. 自分たちのSupabaseプロジェクトを作成する
2. こちらで用意したSQLを実行する
3. アプリ側のSupabase URL / anon keyを差し替える
4. likes / comments / reactions などの課題に取り組む

上級コースは、FlutterだけでなくバックエンドやDBにも触れたいチーム向けの発展課題として扱う。
