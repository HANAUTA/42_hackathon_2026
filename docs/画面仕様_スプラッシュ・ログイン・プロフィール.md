# 画面仕様：スプラッシュ・ログイン・プロフィール入力

---

## 1. スプラッシュ画面

### 役割

アプリ起動時に表示し、ログイン状態を確認して適切な画面へ遷移する。

### 遷移ロジック

```
起動
 ├── 未ログイン → ログイン画面
 └── ログイン済み → ホーム画面
```

---

## 2. ログイン画面

### 認証方式

| 方式 | 実装 |
|------|------|
| メールログイン | Supabase Auth（メール＋パスワード） |
| Googleログイン | Supabase Auth + google_sign_in パッケージ |

### 遷移

- 認証成功 → プロフィール未登録の場合はプロフィール入力画面へ
- 認証成功 → プロフィール登録済みの場合はホーム画面へ

---

## 3. プロフィール入力画面（初回登録）

### 表示タイミング

初回ログイン時のみ表示。`lib/features/auth/profile_screen.dart`。

### 入力項目

| 項目 | 必須 |
|------|------|
| 名前 | ○ |

### 登録後

- usersテーブルにレコード作成（insert）
- ホーム画面へ遷移

---

## 4. プロフィール画面（マイページ／必須課題）

初回登録とは別の**新規画面**。ログインユーザーが自分のプロフィールを見て、名前を編集できるようにする。

### 作る機能

- 自分のプロフィール（名前・自己紹介）を表示する
- **名前を編集**して保存できる（自己紹介も任意で）
- ログアウトできる

※ アイコンは必須ではない（今回は対応しない）。

### 使うDB（作成済み・SQLは触らない）

`users` テーブルを使う。以下のカラムは追加済み。

| カラム | 用途 |
|--------|------|
| `name` | 名前（編集対象） |
| `bio` | 自己紹介（任意・編集対象） |
| `updated_at` | 更新日時（UPDATE時に自動更新。アプリからは渡さない） |

- 取得: `users` を `id = 自分のuserId` で select
- 更新: `users` を `id = 自分のuserId` で update（`name` / `bio`）
- 権限: RLS `users_update_self`（本人のみ更新可）設定済み。追加不要

### やること

- 新規画面を1つ作り、そこに上記の機能を入れる
- ルーティング（`core/router.dart` への追加）と、ホームからの導線も自分で行う
  - ※ `/profile` は初回登録画面で使用済みなので別のパスにすること
